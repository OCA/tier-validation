🛡️ Base Tier Validation - Constraints

Add business constraints validation before tier validation request

📋 Overview

This module extends the base tier validation system to add configurable business constraints that must be satisfied before allowing a validation request. It helps enforce business rules and data quality requirements at the validation request stage.

✨ Features

  • Block or warn users when trying to request validation if conditions are not met
  • Five different constraint types for various business needs
  • Configurable custom error messages per constraint
  • Option to block validation request or only show warning
  • Seamless integration with existing tier validation system
  • Works with any model that uses tier validation

🔧 Constraint Types

Required Field

Ensure a specific field is filled before requesting validation.

Minimum Amount

Ensure a monetary/float field meets a minimum threshold value.

Maximum Amount

Ensure a monetary/float field doesn't exceed a maximum value.

Domain Constraint

Ensure the record matches a specific Odoo domain expression.

Python Code

Custom Python logic for complex business rules and validations.

📚 Usage Examples

Example 1: Required Field

Use case: Ensure customer is selected before validation

  • Constraint Type: Required Field
  • Required Field: partner_id
  • Error Message: "Please select a customer before requesting validation"

Example 2: Minimum Amount

Use case: Block validation for orders below 1000

  • Constraint Type: Minimum Amount
  • Amount Field: amount_total
  • Minimum Amount: 1000.00
  • Error Message: "Orders below 1000 cannot be validated"

Example 3: Python Code

Use case: Complex validation requiring partner and amount > 5000

  • Constraint Type: Python Code
  • Python Code: result = rec.partner_id and rec.amount_total > 5000
  • Error Message: "Validation requires a customer and amount over 5000"

🎯 Use Cases

  • Purchase Orders: Block validation if total amount exceeds budget
  • Invoices: Require payment terms before requesting approval
  • Sales Orders: Ensure delivery date is set before validation
  • HR Requests: Block validation if required documentation is missing
  • Custom Workflows: Implement any complex business validation logic

⚙️ Configuration

  1. Go to Settings > Technical > Tier Validations > Tier Definitions
  2. Open or create a Tier Definition
  3. Navigate to the Constraints tab
  4. Select a constraint type and configure the parameters
  5. Set whether to block the request or only warn
  6. Optionally customize the error message
